## Architecture decisions:

### nDSM adjustment for base elevation override
The nDSM (= DSM - DTM) is pre-computed. When the user changes the base elevation by δ, the volume engine adjusts: `nDSM_adjusted = nDSM - δ` before computing metrics. This correctly models "the ground is δ meters higher than estimated." The `recompute_all_heaps` function opens the nDSM file once (O(1) I/O) and reuses `_compute_metrics_for_mask` for each heap.

### ΔV approximation tolerance
The client-side formula `ΔV ≈ -δ × Σ planimetric_area` gives 11.6% error vs precise recalculation because pixels near the 0.5m height threshold get excluded nonlinearly when nDSM decreases. The test allows 15% tolerance. The formula is still valuable for instant slider feedback (correct direction, correct order of magnitude).
