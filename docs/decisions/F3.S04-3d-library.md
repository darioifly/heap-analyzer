# F3.S04 — 3D Library Decision

**Chosen**: potree-core@2.0.15 + three@0.184.0
**Date**: 2026-04-17

**Reasoning**:
- potree-core is npm-native (clean ESM import), supports Potree 2.0 format (has `loading2/` directory for v2 metadata), and integrates with a standard Three.js Scene/Renderer/Camera.
- Peer dependency `three >0.125.0` is very flexible — allows using latest Three.js (0.184.0) unlike @pnext/three-loader which pins `~0.160.0`.
- Recently maintained (last npm publish 2026-04-08), active GitHub repo (github.com/tentone/potree-core).
- Built-in PointColorType enum includes RGB, HEIGHT/ELEVATION, CLASSIFICATION — covers our 3 color modes.
- `Potree.loadPointCloud(url, baseUrl)` returns `PointCloudOctree` (extends Object3D) — drops into Three.js scene directly.

**Rejected alternatives**:
- @pnext/three-loader@1.0.0 — pinned to three ~0.160.0 (old, conflicts with latest Three.js), version 1.0.0 suggests early/stale development.
- Potree viewer (embedded) — self-contained build artifact, not npm-friendly, fights React DOM lifecycle, requires manual script loading via Vite public/.

**Risks / mitigations**:
- potree-core may have subtle incompatibilities with PotreeConverter 2.1.2 output — mitigated by testing against our synthetic dataset's Potree output.
- GPU memory management is manual — mitigated by implementing explicit dispose() on unmount and toggle stress testing.
